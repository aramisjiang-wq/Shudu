## 📝 文档维护规范

### 更新流程

1. **修改文档** - 更新内容
2. **更新时间戳** - 修改 `Last updated` 字段
3. **同步索引** - 更新本文件的链接和说明
4. **提交变更** - 提交到 Git

### 文档命名规范

- **全大写** - 重要文档（README.md, PRD.md）
- **首字母大写** - 指南类文档（Development_Guide.md）
- **小写下划线** - 配置文件（vercel.json, package.json）

### 文档格式

- 使用 Markdown 格式
- 包含目录（TOC）
- 使用表格和列表提高可读性
- 添加代码示例和截图
